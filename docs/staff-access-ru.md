# Доступ под admin и moderator

## Admin

В текущей базе уже есть стандартный admin-аккаунт:

- email: `admin@audioplatform.com`
- пароль: `admin123`

Если этот аккаунт не был переопределён вручную на сервере, под ним можно
войти прямо через обычную форму логина на сайте.

## Moderator

Отдельный moderator-аккаунт по умолчанию сейчас не создаётся автоматически.
Рабочий путь такой:

1. Зарегистрировать обычного пользователя через сайт.
2. На сервере поменять ему роль в PostgreSQL на `moderator`.

Пример команды на сервере:

```bash
cd /root/resonans-sound/infra
docker compose exec -T postgres \
  psql -U "$POSTGRES_USER" -d "$POSTGRES_DB" \
  -c "update users set role='moderator' where email='user@example.com';"
```

После этого:

- выйдите из текущей сессии, если пользователь уже был залогинен
- зайдите снова через обычную форму логина
- в `GET /api/v1/users/me` и в UI роль уже будет видна как `moderator`

## Что сейчас умеют staff-роли

На текущем этапе:

- `admin` и `moderator` могут удалять любые треки
- ручная moderation-публикация сейчас не является обязательным шагом
- основной продуктовый flow сейчас идёт через автопубликацию после processing
